총 **19주** ≈ 4.5개월 (v1, 모바일 제외). 병렬화 가능 포인트:

- M3 (schema) 은 M2 후반부부터 병행 가능 → **2주 단축**
- M5 (툴바) 와 M6 (표/이미지) 는 팀이 2명 이상이면 병렬 → **2주 단축**

## 각 마일스톤 Exit criteria (체크리스트)

### M0

- [ ] `pnpm install && pnpm test` 그린
- [ ] CI 파이프라인 3단계 (lint / test / build) 녹색
- [ ] 공식 스펙 PDF 확보

### M1

- [ ] 10종 픽스처 파싱 성공
- [ ] `HwpxDocument.sections[].body` 에서 텍스트 추출 정확도 100%
- [ ] 단위 테스트 커버리지 ≥ 80% (hwpx-codec)

### M2

- [ ] 원본 → 직렬화 → 한컴오피스 열기 성공 (10/10)
- [ ] `parse ∘ serialize ∘ parse` 의미 동일성 (property test 1000케이스)
- [ ] `opaque` 보존 검증

### M3

- [ ] toPM(fromPM(x)) 안정적
- [ ] 서식 upsert 후 header refList 무결성

### M4

- [ ] 에디터에서 편집한 후 저장 → 한컴오피스 열림
- [ ] 한글 IME 입력 1000자 안정

### M5

- [ ] 툴바 기능 전체 E2E 녹색
- [ ] 스타일 변경이 header refList 에 반영

### M6

- [ ] 표 편집 (10행×10열 기준) 모든 연산 녹색
- [ ] 이미지 5MB 삽입/크기조정/저장 OK

### M7

- [ ] 1MB 로드 < 1초 (p50)

- [ ] TypeDoc / 사용자 가이드 배포

## 체크포인트 주기

- 매주 금요일 오후: 진행 리뷰 (계획 대비 실적, 차단 이슈)
- 각 마일스톤 종료일: Demo + Exit criteria 검증
- 매 2주: 외부 샘플(공공 HWPX 파일 세트)로 정합성 테스트

## 자원 투입 (제안)

- 1인 풀타임 시: 약 4.5개월 타임라인, 풀스택으로 진행
- 2인: 코덱/스키마 전담 1명 + 에디터/플랫폼 1명 → 3.5개월
- 3인 이상: 추가 인원은 QA/픽스처 큐레이션, 접근성, 문서화, 데스크톱 사이닝/배포에 투입
