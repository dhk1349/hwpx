# 프로젝트 범위

## 비전

브라우저에서 HWPX(한글 XML 포맷) 문서를 **열고, 편집하고, 저장**할 수 있는 웹 기반 에디터를 만든다. 서버 없이도 동작하는 클라이언트-퍼스트 구조로, 한컴오피스가 없는 환경/OS에서도 HWPX를 일급 시민으로 다룰 수 있게 한다.

## 목표 (Goals)

1. **스펙 정확성**: KS X 6101 (OWPML) 스펙을 준수하여, 한컴오피스가 읽을 수 있는 유효한 HWPX를 생성/수정
2. **핵심 편집 UX**: MS Word / Google Docs 급의 편집 경험 (텍스트, 서식, 표, 이미지, 리스트)
3. **왕복 보존**: 에디터가 지원하지 않는 요소를 파괴하지 않고 **원본을 보존**한 채 라운드트립
4. **오픈소스**: MIT / Apache-2.0 라이선스로 공개, 문서화된 API 제공
5. **테스트 가능**: 파서/직렬화/에디터 각 계층에 대한 자동화된 테스트

## 비목표 (Non-goals, v1 기준)

- HWP 5.0 **바이너리** 포맷 지원 (별도 어댑터로 추후 검토)
- 실시간 협업 편집(CRDT / OT) — 아키텍처만 확장 가능하게 설계
- 인쇄 미리보기의 픽셀 단위 한컴오피스 일치 — 표시는 "합리적으로 근사"까지만
- 암호화된 HWPX 의 암복호화
- HWPX 내장 스크립트 실행 (보안상 비활성)
- HWPX 에만 있는 희귀 고급 기능 (고급 수식 편집 UI, OLE 개체 편집 등)

## 핵심 사용 시나리오 (User Stories)

1. **U1**: 사용자가 `.hwpx` 파일을 브라우저에 드래그&드롭하면 1초 이내에 본문이 렌더링된다
2. **U2**: 사용자가 텍스트를 선택하고 **굵게/기울임/밑줄/색상/크기** 를 변경할 수 있다
3. **U3**: 사용자가 **문단 정렬, 들여쓰기, 줄 간격, 스타일 적용** 을 할 수 있다
4. **U4**: 사용자가 **표를 삽입/편집**(행·열 추가·삭제·병합·분할, 셀 편집)할 수 있다
5. **U5**: 사용자가 **이미지를 삽입/크기조절/정렬**할 수 있다 (붙여넣기 포함)
6. **U6**: 사용자가 **순서/비순서 리스트** 를 만들 수 있다
7. **U7**: 사용자가 수정한 문서를 `.hwpx` 로 **다운로드** 하고, 해당 파일을 한컴오피스/뷰어가 **오류 없이 열 수 있다**
8. **U8**: 사용자가 `Ctrl+Z` / `Ctrl+Y` 로 되돌리기/다시 실행 할 수 있다
9. **U9**: 지원하지 않는 요소(예: 복잡한 도형)가 있어도 저장 시 **원본이 보존** 된다

## 측정 가능한 성공 지표

| 지표                                       | v1 목표                                        |
| ------------------------------------------ | ---------------------------------------------- |
| 평균 로드 시간 (1MB hwpx)                  | < 1.0초                                        |
| 왕복 보존율 (열기→저장 후 의미론적 diff 0) | ≥ 95% 의 공공 샘플                             |
| 단위 테스트 라인 커버리지                  | ≥ 80%                                          |
| 주요 요소 지원                             | 문단/런/표/이미지/리스트/스타일 = 100%         |
| 한컴오피스 열기 성공률                     | 100% (생성한 파일이 열리지 않으면 릴리스 차단) |

## 참고: 확장(차기 버전) 후보

- HWP 5.0 바이너리 → HWPX 변환 (openhwp 의 IR 차용)
- PDF / DOCX 내보내기
- 실시간 협업 (Yjs / Automerge)
- 플러그인 시스템 (수식, 차트)
- 로컬 자동 저장 (IndexedDB)
